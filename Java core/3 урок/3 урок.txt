Инкапсуляция позволит скрыть детали реализации, и открыть только то что необходимо в последующем использовании.
Другими словами инкапсуляция – это механизм контроля доступа.

Инкапсуляция - это один из принципов объектно-ориентированного программирования, 
который позволяет скрыть внутреннюю реализацию объекта и предоставить доступ к его состоянию
только через определенные публичные методы (геттеры и сеттеры). 

Основная идея инкапсуляции заключается в том, чтобы обеспечить контролируемый доступ к данным объекта и
предотвратить неправильное использование или изменение внутреннего состояния объекта извне.


Данные объекта можн ополучить только через специальные методы геттеры
Установить данные можно только через сепциальные методы сеттеры


1. Модификаторы доступа
а. public - доступен из любой точки файловой системы
б. private - доступен только в своем классе
в. protected - доступен в своем пакете

2. Создать переменные обьекта каждым модификатором.

3. Геттеры, сеттеры. Создание переменных объекта через геттеры

4. Прописать условие в сеттере
