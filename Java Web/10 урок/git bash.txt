Генерация нового SSH ключа в Git Bash
Откройте Git Bash.
1.Сгенерируйте новый SSH ключ, используя свой email. Если у вас уже есть SSH ключ, вы можете пропустить этот шаг и перейти к добавлению существующего ключа на GitHub.

ssh-keygen -t ed25519 -C "ваш_email@example.com"
При создании ключа вас попросят ввести место для сохранения файла ключа. Можно просто нажать Enter, 
чтобы использовать местоположение по умолчанию. После этого вас могут попросить ввести парольную фразу для дополнительной защиты ключа.


2. Добавление SSH ключа в ssh-agent
Запустите ssh-agent в фоновом режиме.

eval "$(ssh-agent -s)"
Добавьте ваш SSH приватный ключ в ssh-agent. Если вы создали ключ с другим именем или если вы хотите добавить существующий ключ, 
замените id_ed25519 на имя вашего файла приватного ключа.

ssh-add ~/.ssh/id_ed25519



3. Добавление SSH ключа на GitHub
Скопируйте содержимое вашего публичного ключа в буфер обмена. Для этого можно использовать команду cat в Git Bash, а затем скопировать вывод вручную.

cat ~/.ssh/id_ed25519.pub


Перейдите на GitHub и войдите в свой аккаунт.
Нажмите на ваш аватар в правом верхнем углу и перейдите в "Settings".
В левом меню выберите "SSH and GPG keys".
Нажмите на кнопку "New SSH key" или "Add SSH key".
В поле "Title" введите описательное название для ключа.
В поле "Key" вставьте содержимое вашего публичного ключа из буфера обмена.
Нажмите "Add SSH key" для сохранения нового ключа.
