Дополнение к 1 уроку:

1. request и response. 
Являются ключевыми понятиями. Они используются для обработки входящих запросов от клиентов
(например, браузеров или других клиентских приложений) и отправки ответов обратно.

request(запрос) -это сообщение, отправленное клиентом серверу.
Оно содержит информацию о том, что клиент хочет сделать или получить.

responce(ответ) - это сообщение, отправленное сервером обратно клиенту в ответ на его запрос.


2. Сервлеты в Java EE (Enterprise Edition) — это Java классы, 
которые обрабатывают запросы клиентов (например, браузеров) на сервере и генерируют ответы. 
Сервлеты являются основным строительным блоком веб-приложений на Java EE.

3. RESTful API – это набор принципов и ограничений для создания веб-сервисов
	методы:

1.GET для чтения данных.
2.POST для создания новых ресурсов.
3.PUT для обновления существующих ресурсов.
4.DELETE для удаления ресурсов.
5.PATCH используется для частичного обновления существующего ресурса.
В отличие от метода PUT, который предполагает полное обновление ресурса, 
PATCH применяется для внесения частичных изменений.
